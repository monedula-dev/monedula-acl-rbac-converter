# Examples

Each subdirectory is a self-contained, runnable example of one end-to-end flow
path through `monedula-acl-rbac`. The intent: pick the one that matches how *your*
ACLs are stored, read its `README.md`, run its `run.sh`, and see exactly **what
ACLs convert to what RBAC bindings** â€” with committed `expected/` outputs you can
diff against.

All of them (except `cfk-manifests`, see below) describe the same realistic
"microservices platform" scenario â€” orders / payments / analytics services, a
prefixed reader, an `All`-on-topic owner, a **read-write relay** that both
consumes and produces one topic, and a DENY that RBAC cannot represent â€”
expressed through a different input source each time. The read-write relay
(`svc-orders-relay`, Read+Write+Describe on `orders`) is the interesting case:
no single `Developer*` role grants both read and write, so it converts to **two**
bindings (`DeveloperRead` + `DeveloperWrite`) that together reproduce its access.

`cfk-manifests` is the exception: a CFK `Kafka` CR expresses ACLs only through
cluster-wide `superUsers` (â†’ `SystemAdmin`), so it carries a smaller
superuser-plus-existing-binding scenario rather than per-principal topic ACLs.

**Start with [`live-mds-migration/`](live-mds-migration/)** for the full picture:
it runs the entire migration â€” including `apply`, effective-access `verify`, and
`delete-acls` â€” against a real Confluent stack stood up with docker-compose.

The file-based examples need only a `monedula-acl-rbac` binary (`make build`, or
they build one for you). Each also ships a `docker-compose.yml` so you can run it
with no local toolchain:

```sh
cd examples/text-dump
./run.sh --check                              # uses a local/just-built binary
# or, with Docker only:
docker compose run --rm converter ./run.sh --check
```

`run.sh --check` regenerates the artifacts into `out/` and diffs them against the
committed [`expected/`](text-dump/expected/) goldens, so the examples double as a
smoke test that the tool still produces what the docs claim.
